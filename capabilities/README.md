# `capabilities/` — the kinetic contracts

This folder holds the ontology's **kinetic layer**: the **actions** (create / modify / delete objects,
set properties, write links) and **functions** (compute readings, no write-back) declared in
`../ontology.md §2`. One file per contract.

- `actions/` — source · screen · underwrite · close · transform · talent · retire · finance
- `functions/` — monitor · capital-allocation · portfolio-impermanence · product-users

## Do not conflate the two senses of "capability"
- A **capability** in the composable-competency sense (`../foundations/org-as-an-intelligence.md §3`) is an
  *invocable, composable* lever — today the operator levers **source · screen · underwrite · close · transform · hire**.
- These **contracts** are the *kinetic machinery* that **realizes** those capabilities — plus the mechanical
  actions (`retire`, `finance`) and the read-only **functions**, which are not "levers".

Same machinery, two grains: the **competency** (capability) vs the **kinetic contract** (action / function)
that implements it. The folder name and the `list_capabilities` tool are historical; what they *hold* is the
kinetic layer. Never equate a capability with an action.
